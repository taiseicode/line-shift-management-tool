# LINEシフト管理ツール

## 概要
LINEとLIFFを活用して、シフト提出から管理までを一元化するWebアプリケーションです。  
アルバイト先での実際の運用課題をもとに開発しました。
従来のシフト管理では、提出方法が口頭・LINE・紙などに分散しやすく、未提出者の把握や不足人数の確認、集計作業に時間がかかる問題がありました。  
本アプリでは、スタッフはLINE上からシフトを提出でき、管理者はブラウザ上の管理画面から提出状況や不足人数を確認できます。

## 背景
アルバイト先では、以下のような課題がありました。

- シフト提出がバラバラで管理しづらい
- 誰が未提出なのか分かりにくい
- 必要人数に対して不足している日をすぐ把握できない
- 管理者が手作業で集計する負担が大きい

これらを解決するために、**LINEで提出できること**と**管理画面で見やすく集計できること**を重視して開発しました。

## 主な機能

### スタッフ向け機能
- LINE / LIFFからシフト提出
- 出勤 / 休みの登録
- 勤務時間の入力
- 提出済みシフトの確認
- 締切後の編集・削除制御

### 管理者向け機能
- 管理画面で日別シフト一覧を確認
- 必要人数の設定
- 不足人数の自動計算と可視化
- Excel出力
- ユーザー一覧管理
- ユーザーの有効 / 無効切り替え
- ユーザー名変更
- 締切設定

---

## 工夫した点

### 1. 実運用を意識した設計
単に「提出できる」だけでなく、現場で使うことを想定し、未提出者や不足人数が一目で分かる管理画面を実装しました。

### 2. 権限を意識したユーザー管理
ユーザー名の変更や有効 / 無効の切り替えを管理者側で制御できるようにし、現場運用に耐えられる設計を意識しました。

### 3. 締切ロジックの実装
提出締切を過ぎた場合、スタッフ側から変更・削除できないようにし、シフト回収のルールをシステム側で担保しました。

### 4. 責務ごとの分割
当初は1ファイルにまとまっていた構成を、`routes` / `services` / `repositories` に分割し、可読性と保守性を改善しました。

### 5. LINE連携
一般的なCRUDアプリではなく、LINE Messaging API と LIFF を組み合わせることで、利用者が普段使うLINE上から操作できるようにしました。
