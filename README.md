# LINEシフト管理ツール

## 概要

LINE × LIFF を活用し、アルバイトのシフト提出・管理を効率化するWebアプリケーションです。

従来のシフト管理では、提出方法が口頭・LINE・紙などに分散し、

* 未提出者の把握が困難
* 必要人数の不足確認に時間がかかる
* 管理者の集計負担が大きい

といった課題がありました。

本アプリでは、

* スタッフはLINE上からシフトを提出
* 管理者はブラウザ上で提出状況・不足人数を一元管理

することで、シフト管理業務の効率化を実現しています。

---

## 背景

アルバイト先で実際に運用されていたシフト管理に課題を感じ、開発しました。

* シフト提出方法が統一されていない
* 未提出者が分かりにくい
* 必要人数との差分が即座に把握できない
* 手作業での集計に時間がかかる

これらを解決するために、「LINEで簡単に提出できること」と「管理画面で一目で把握できること」を重視しました。

---

## 主な機能

### スタッフ向け

* LINE / LIFFからシフト提出
* 出勤 / 休みの登録
* 勤務時間入力（時間帯指定）
* 提出済みシフトの確認
* 締切後の編集・削除制御

### 管理者向け

* 日別シフト一覧の表示
* 必要人数の設定
* 不足人数の自動計算・可視化
* ユーザー管理（有効 / 無効・名前変更）
* 締切設定
* Excel出力機能

---

## 工夫した点

### 1. 実運用を意識した設計

現場で使うことを想定し、未提出者や不足人数を直感的に把握できるUIを実装しました。

### 2. 締切ロジックの実装

提出締切を過ぎた場合、スタッフ側から変更・削除できないようサーバー側で制御しています。
これにより、運用ルールをシステム側で担保しています。

### 3. ユーザー管理機能

管理者側でユーザーの有効 / 無効切り替えや名前変更を可能にし、現場運用に対応しました。

### 4. 責務ごとの設計分割

初期は単一ファイル構成でしたが、

* routes
* services
* repositories

に分割し、可読性・保守性を向上させました。

### 5. LINE連携（LIFF）

LINE Messaging API と LIFF を組み合わせることで、ユーザーが普段使うLINE上から操作できる設計にしました。
